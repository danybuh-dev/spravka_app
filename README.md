# Справка по делу

Локальное HTML-приложение с Python backend для подготовки справки о ходе рассмотрения дела на основании загруженных материалов.

## Что умеет

- строит сценарий ровно в 3 шага: загрузка документов, проверка автозаполнения, выгрузка справки;
- сначала предлагает загрузить процессуальный документ пользователя и определение суда о назначении дела к слушанию;
- автоматически извлекает текст из читаемых `PDF` и `DOCX`;
- ищет стороны в шапке процессуального документа или в начале судебного акта;
- ищет председательствующего, дату и время заседания прежде всего в определении суда о назначении дела к слушанию;
- автозаполняет реквизиты дела, сведения о заседании и базовые выводы по позиции;
- заполняет раздел `Перспектива рассмотрения спора` по просительной части жалобы, отзыва или возражений;
- формирует `Обоснование позиции` в формате: тезис и один раскрывающий абзац;
- умеет отправлять весь массив загруженных документов в OpenAI API для анализа GPT;
- показывает индикатор статуса GPT: зеленый при успешной работе, красный если GPT не подключен или не подтвержден;
- получает от модели структурированные данные и готовую справку в конечной форме;
- формирует обязательные разделы: информация о деле, выводы о перспективах, описание ситуации, обоснование перспектив;
- учитывает статус спора, дату и время заседания, председательствующего и подробный состав сторон;
- выделяет ключевые фрагменты для краткой справки на 2-3 страницы;
- формирует черновик справки;
- позволяет после проверки данных одной кнопкой выгрузить справку в `.pdf` и `.docx`.

## Как запустить

1. Задайте ключ OpenAI в окружении:

```bash
cp .env.example .env
```

2. Заполните `.env`:

```bash
OPENAI_API_KEY="sk-..."
```

3. Запустите локальный сервер:

```bash
./start.sh
```

4. Откройте приложение в браузере:

[http://127.0.0.1:8000](http://127.0.0.1:8000)

Для работы разбора `PDF/DOCX`, экспорта `PDF/DOCX` и GPT-анализа странице нужен доступ к CDN и к OpenAI API.

## Ограничения текущей версии

- поддерживаются текстовые файлы (`.txt`, `.md`, `.html`, `.json`), текстовые `PDF` и `DOCX`;
- сканированные PDF без текстового слоя не разбираются;
- старый формат `.doc` не поддерживается автоматически;
- итоговый текст строится по эвристикам, поэтому требует юридической вычитки;
- точность автозаполнения и ответа GPT зависит от структуры исходных документов;
- ключ OpenAI должен храниться в окружении backend-сервера;
- `.env` не должен попадать в git или в общие архивы;
- для production всё равно стоит добавить отдельную авторизацию, логирование и ограничения доступа к backend.
